## User actions to fully activate

### 1. Cal.com webhook (3 minutes)
```bash
# In Cal.com: Settings → Developer → Webhooks → Create webhook
#   Subscriber URL: https://publicpulse.com.bd/api/webhooks/cal-com
#   Subscribe to:   BOOKING_CREATED, BOOKING_RESCHEDULED, BOOKING_CANCELLED
#   Copy the "Secret" Cal.com generates.

sst secret set CAL_WEBHOOK_SECRET "<paste secret from cal.com>" --stage production
AWS_PROFILE=eventpulse npx sst deploy --stage production
```
After this: every new booking auto-creates a lead in /manage/leads + fires Meta CAPI.

### 2. Resend webhook (3 minutes)
```bash
# In Resend: Webhooks → Add endpoint
#   URL: https://publicpulse.com.bd/api/webhooks/resend
#   Subscribe to: email.sent, email.delivered, email.opened, email.clicked,
#                 email.bounced, email.complained
#   Copy the "Signing secret" (starts with whsec_).

sst secret set RESEND_WEBHOOK_SECRET "<paste whsec_... signing secret>" --stage production
AWS_PROFILE=eventpulse npx sst deploy --stage production
```
After this: newsletter_sends gets engagement data; can build a dashboard pulse.
